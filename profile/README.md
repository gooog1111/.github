## Что здесь можно найти

* **Инструменты и утилиты** — небольшие проекты для автоматизации, настройки и обслуживания.
* **OpenWrt и LuCI** — переводы, локализация и материалы для сетевых компонентов.
* **Linux и оборудование** — заметки, инструкции, драйверы, прошивки и практические решения.
* **Исследования и документация** — описания протоколов, форматов, модификаций и технических наблюдений.

## Как пользоваться репозиториями

* В каждом проекте смотрите `README.md`: там обычно описаны назначение, установка и базовое использование.
* Если в репозитории есть отдельная документация, она находится рядом с кодом или указана в README.
* Перед созданием Issue проверьте открытые и закрытые обсуждения: возможно, похожий вопрос уже разбирался.
* Для сообщений об ошибках указывайте окружение, версию, шаги воспроизведения, логи и ожидаемый результат.

## Участие

Любой вклад приветствуется: исправление опечаток, улучшение документации, перевод, сообщение об ошибке, идея или Pull Request.

Перед Pull Request желательно:

* сохранить стиль конкретного репозитория;
* не смешивать несколько независимых изменений в одном PR;
* кратко объяснить, что изменено и зачем;
* приложить проверку или описание тестирования, если это применимо.

## Безопасность

Если вы нашли потенциальную уязвимость, пожалуйста, не публикуйте её в открытом Issue. Следуйте инструкциям `SECURITY.md` или свяжитесь со мной приватно через доступные контакты профиля.

## What You Can Find Here

* **Tools and utilities** — small projects for automation, configuration, and maintenance.
* **OpenWrt and LuCI** — translations, localization, and materials for network components.
* **Linux and hardware** — notes, instructions, drivers, firmware, and practical solutions.
* **Research and documentation** — protocol notes, data formats, modifications, and technical observations.

## How to Use the Repositories

* Start with each project's `README.md`: it usually describes the purpose, installation, and basic usage.
* If a repository has separate documentation, it is stored near the code or linked from the README.
* Before opening an Issue, check open and closed discussions: a similar question may already have been answered.
* For bug reports, include the environment, version, reproduction steps, logs, and expected result.

## Contributing

Any contribution is welcome: typo fixes, documentation improvements, translations, bug reports, ideas, or Pull Requests.

Before opening a Pull Request, please:

* keep the style of the specific repository;
* avoid mixing several unrelated changes in one PR;
* briefly explain what changed and why;
* include a check or testing notes when applicable.

## Security

If you find a potential vulnerability, please do not publish it in an open Issue. Follow `SECURITY.md` or contact me privately through the available profile contact methods.
